# Next Steps on GitHub

The following actions depend on manual configuration in the GitHub web
interface and cannot be completed through repository files alone.

## Recommended Repository Description

Use this short description:

`A custom Zabbix dashboard widget that visualizes numeric item data as a weekly heatmap.`

## Suggested Topics

Add these repository topics in the GitHub UI:

- `zabbix`
- `zabbix-widget`
- `zabbix-dashboard`
- `monitoring`
- `observability`
- `heatmap`
- `docker`
- `logs`
- `devops`
- `sre`

## Manual GitHub Actions

1. Set the repository description using the suggestion above.
2. Add the suggested repository topics.
3. Switch the default branch to `main` if you want to move away from
   `master`.
4. Configure a social preview image for the repository.
   A practical starting point is a cropped version of
   `docs/images/heatmap-example.png`.
5. Create the first GitHub release as `v1.0.0`.
6. Review repository visibility, branch protection, and maintainer access
   rules as needed.
