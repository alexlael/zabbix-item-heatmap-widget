# Contributing

Thank you for your interest in contributing to this project.

## Before You Open a Pull Request

- Check existing issues and pull requests to avoid duplicate work.
- Keep changes focused and scoped to a clear objective.
- Document user-facing changes in `README.md`, `CHANGELOG.md`, or `docs/`
  when relevant.

## Development Notes

- Do not change behavior without explaining the motivation and expected
  outcome.
- Prefer small, reviewable pull requests over large mixed changes.
- If the change affects compatibility, installation, or screenshots, update
  the related documentation.

## Pull Request Expectations

- Describe what changed and why.
- Include verification steps.
- Mention any follow-up work or known limitations.
